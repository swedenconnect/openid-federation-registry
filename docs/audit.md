# Audit Event Documentation

This document details the audit events that can be generated by the `RegistryService` for auditing operations read and 
update. Each event corresponds to a specific operation performed on entities, policies, or trustmarks.

## Event Types

| Event Type                          | Description                                            | Fields                                                                 |
|-------------------------------------|--------------------------------------------------------|------------------------------------------------------------------------|
| `POLICY_CREATE_UPDATED`             | Emitted when a policy is created or updated.           | `extId`, `oldData`, `newData`                                         |
| `POLICY_DELETED`                    | Emitted when a policy is deleted.                      | `extId`, `oldData`                                                    |
| `ENTITY_CREATED_UPDATE`             | Emitted when an entity is created or updated.          | `issuer`, `subject`, `extId`, `oldData`, `newData`                    |
| `ENTITY_DELETED`                    | Emitted when an entity is deleted.                     | `extId`, `oldData`                                                    |
| `TRUSTMARK_SUBJECT_CREATE_UPDATE`   | Emitted when a TrustMark subject is created or updated.| `extId`, `oldData`, `newData`                                         |
| `TRUSTMARK_SUBJECT_DELETED`         | Emitted when a TrustMark subject is deleted.           | `trustMarkId`, `extId`, `oldData`                                     |

---

## Event Field Descriptions

| Field         | Description                                                                 |
|---------------|-----------------------------------------------------------------------------|
| `extId`       | External identifier for the event. Typically a UUID referring to the resource. |
| `oldData`     | JSON representation of the resource's prior state.                          |
| `newData`     | JSON representation of the resource's updated state (if applicable).        |
| `issuer`      | Entity issuing the event (only applicable for entity-related events).        |
| `subject`     | Subject of the event (used for entity-related events).                      |
| `trustMarkId` | Identifier for the affected TrustMark (used for trustmark-related events).   |

---

## Event Use Case Examples

1. **Policy Update**
    - When a policy is updated via the API, the event `POLICY_CREATE_UPDATED` is emitted. It contains both the old and new states of the `PolicyRecord`.

2. **Entity Deletion**
    - When an entity is deleted, the `ENTITY_DELETED` event is emitted with the `extId` of the entity and its previous state.

3. **TrustMark Creation**
    - A TrustMark subject creation event emits `TRUSTMARK_SUBJECT_CREATE_UPDATE`, logging both the old and new states of the TrustMark subject.

---

### Notes
- Events provide detailed auditing information useful for compliance and monitoring purposes.
- Fields like `issuer` and `subject` are reserved for entity-related operations to enrich audit logs with additional context.
